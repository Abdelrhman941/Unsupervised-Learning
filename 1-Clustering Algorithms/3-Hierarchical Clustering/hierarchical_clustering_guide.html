üéØ What is Hierarchical Clustering?

Hierarchical clustering creates a tree-like structure (dendrogram) showing relationships between data points at different levels of granularity. Unlike K-means or DBSCAN, it doesn't require pre-specifying the number of clusters.

üéØ Core Principle: Build a hierarchy of clusters by iteratively merging (agglomerative) or splitting (divisive) clusters based on distance/similarity measures.

üöÄ When to Use Hierarchical Clustering

‚úÖ Ideal Scenarios:

  • Unknown cluster count: Explore different numbers of clusters
  • Hierarchical structure: Data has natural hierarchy
  • Small to medium datasets: < 10,000 points typically
  • Deterministic results: Same input always gives same output
  • Visualization needs: Dendrogram provides insights
  • Domain understanding: Need to understand cluster relationships

‚ùå Avoid Hierarchical Clustering When:

  • Large datasets: O(n¬≥) time complexity is prohibitive
  • Real-time applications: Too slow for live clustering
  • Memory constraints: Requires O(n¬≤) memory
  • Noisy data: Sensitive to outliers
  • High dimensions: Distance becomes meaningless

üí™ Advantages vs Limitations

‚úÖ Advantages

  • No Cluster Count Needed: Explore different numbers of clusters
  • Deterministic: Same input always produces same result
  • Hierarchical Insights: Dendrogram shows cluster relationships
  • Any Distance Metric: Works with various distance measures
  • No Assumption on Shape: Can find various cluster shapes

‚ùå Limitations

  • Computational Complexity: O(n¬≥) time, O(n¬≤) space
  • Sensitive to Outliers: Can distort entire hierarchy
  • Irreversible Decisions: Cannot undo merge/split operations
  • Difficulty with Large Data: Doesn't scale well
  • Parameter Selection: Choosing linkage and distance can be tricky

üîÑ Two Main Approaches

üìä Clustering Strategies:

1. Agglomerative (Bottom-Up) ‚Äì Most Common
  • Start with each point as its own cluster
  • Iteratively merge closest clusters
  • Continue until all points form a single cluster
  • Memory: O(n¬≤) space complexity
2. Divisive (Top-Down) ‚Äì Less Common
  • Start with all points in one cluster
  • Iteratively split clusters into smaller groups
  • Continue until each point stands alone
  • Complexity: More computationally expensive
